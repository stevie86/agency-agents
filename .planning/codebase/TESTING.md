# Testing Patterns

**Analysis Date:** 2026-03-29

## Test Framework

**Runner:**
- Custom bash script: `scripts/lint-agents.sh`
- Config: No separate config file; script contains validation rules
- Language: Bash

**Validation Commands:**
```bash
# Lint all agents
./scripts/lint-agents.sh

# Lint specific files
./scripts/lint-agents.sh engineering/engineering-frontend-developer.md

# Run same check as CI (on changed files)
git diff --name-only origin/main...HEAD -- '**/*.md' | xargs ./scripts/lint-agents.sh
```

## Test File Organization

**Location:**
- Validation script: `scripts/lint-agents.sh`
- CI workflow: `.github/workflows/lint-agents.yml`
- Integration scripts: `scripts/convert.sh`, `scripts/install.sh`

**Structure:**
```
scripts/
├── lint-agents.sh      # Validation script
├── convert.sh          # Integration file generator
└── install.sh          # Tool installer
.github/workflows/
└── lint-agents.yml     # CI workflow
```

## Test Structure

**Lint Script Validation Rules:**
```bash
# 1. Check frontmatter delimiters
# 2. Check required frontmatter fields (name, description, color)
# 3. Check recommended sections (warn only)
# 4. Check file has meaningful content (≥50 words)
```

**CI Workflow Steps:**
```yaml
steps:
  - Checkout repository
  - Get changed agent files
  - Run agent linter on changed files
```

## Validation Rules

**Frontmatter Validation (ERRORS):**
1. Must start with `---` on line 1
2. Must include `name:` field
3. Must include `description:` field  
4. Must include `color:` field
5. Frontmatter must be properly closed with `---`

**Content Validation (WARNINGS):**
1. Missing recommended sections: "Identity", "Core Mission", "Critical Rules"
2. Body content shorter than 50 words

**Example Validation Output:**
```
Linting 147 agent files...

ERROR engineering/engineering-frontend-developer.md: missing frontmatter field 'color'
WARN  design/design-whimsy-injector.md: missing recommended section 'Identity'
WARN  testing/testing-evidence-collector.md: body seems very short (< 50 words)

Results: 1 error(s), 2 warning(s) in 147 files.

FAILED: fix the errors above before merging.
```

## CI/CD Integration

**GitHub Actions Workflow (`.github/workflows/lint-agents.yml`):**
- **Trigger:** Pull requests that modify files in agent directories
- **Directories monitored:** All 12 agent directories
- **Process:**
  1. Checkout code with full history
  2. Identify changed `.md` files in agent directories
  3. Run `scripts/lint-agents.sh` on changed files only
  4. Fail if any errors found

**Workflow Configuration:**
```yaml
on:
  pull_request:
    paths:
      - 'design/**'
      - 'engineering/**'
      - 'game-development/**'
      # ... all agent directories
```

## Pre-Commit Checklist

**Before Submitting a PR:**
1. **Run linter locally:** `./scripts/lint-agents.sh`
2. **Follow agent template structure** (see CONTRIBUTING.md)
3. **Include personality and voice** (not generic)
4. **Add concrete code/template examples** (2-3 minimum)
5. **Define success metrics** (with specific numbers)
6. **Include step-by-step workflow**
7. **Proofread** for typos, formatting issues, clarity
8. **Test in real scenarios** (use the agent)

**Quick Validation Commands:**
```bash
# Lint your specific file
./scripts/lint-agents.sh your-agent.md

# Lint all agents to ensure nothing else broke
./scripts/lint-agents.sh

# Generate integration files (optional but recommended)
./scripts/convert.sh --parallel

# Test installation (optional)
./scripts/install.sh --tool opencode
```

## Common Validation Errors and Fixes

**Error: `missing frontmatter opening ---`**
- Fix: Ensure file starts with `---` on line 1

**Error: `missing frontmatter field 'name'`**
- Fix: Add `name: Agent Name` to frontmatter

**Error: `empty or malformed frontmatter`**
- Fix: Ensure frontmatter section is properly formatted with `---` delimiters

**Warning: `missing recommended section 'Identity'`**
- Fix: Add a section with "Identity" in the header (e.g., "🧠 Your Identity & Memory")

**Warning: `body seems very short (< 50 words)`**
- Fix: Expand agent description, add more content about capabilities

## Integration File Generation

**Purpose:** Convert agent files to tool-specific formats (`.mdc`, `SKILL.md`, etc.)

**Commands:**
```bash
# Generate all integration files (serial)
./scripts/convert.sh

# Generate in parallel (faster)
./scripts/convert.sh --parallel

# Generate for specific tool only
./scripts/convert.sh --tool cursor
./scripts/convert.sh --tool opencode

# Dry-run to preview changes
./scripts/convert.sh --dry-run
```

**Important:** Never edit files in `integrations/` directory - they are generated by `convert.sh`.

## Installation Testing

**Purpose:** Install agents to local tool configurations

**Commands:**
```bash
# Interactive installer (auto-detects tools)
./scripts/install.sh

# Install for specific tool
./scripts/install.sh --tool cursor
./scripts/install.sh --tool opencode

# Non-interactive install for all detected tools
./scripts/install.sh --no-interactive --parallel
```

## Test Coverage

**Requirements:** No formal test coverage requirements, but linting ensures:
- Frontmatter validity
- Content quality (word count)
- Recommended section presence

**View Coverage:**
- Manual inspection of lint script results
- CI workflow provides automated coverage for PRs

## Test Types

**Unit Tests:**
- Not applicable (no code to unit test)

**Integration Tests:**
- Lint script validates individual agent files
- CI workflow validates changed files in PR context

**E2E Tests:**
- Manual testing of agents in real AI tools
- Installation script testing with actual tools

## Common Patterns

**Async Testing:**
- Not applicable (bash scripts run synchronously)

**Error Testing:**
- Lint script returns exit code 1 on errors, 0 on success
- CI workflow fails PR if lint script fails
